# Security Policy

`noesis-tinymux` is a pre-production integration laboratory and currently makes
no claims of production readiness.

Do not report secrets, credentials, private player data, private world content,
production transcripts, or production logs in public issues.

If GitHub private vulnerability reporting is available for this repository, use
it for security-sensitive reports. Otherwise, contact the repository owner
privately before public disclosure.

Useful reports should include:

- the affected revision;
- reproduction steps;
- expected and observed behavior;
- impact;
- sanitized evidence.

Production credentials, transcripts, logs, private server addresses, deployment
inventory, player data, and private world content must never be attached to a
public report.
